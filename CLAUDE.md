# 백층 던전 (Dungeon 100)

매판 새로 생성되는 던전을 뚫고 100층까지 내려가는 **탑다운 3D 로그라이크** (three.js + react-three-fiber). 모바일 우선 웹앱 (React + Vite + TypeScript).
**공개 저장소** — 시크릿·내부 경로·**제출(행사) 관련 사항 커밋 금지** (2026-07-22 분리): 제출 체크리스트·소개/기술 문서·촬영 가이드는 비공개 저장소 **[dungeon100-submission](https://github.com/Hakhyun-Kim/dungeon100-submission)** 에서만 관리하고, 이 저장소의 문서·README에는 제출 로지스틱스를 적지 않는다.
설계·로드맵: `docs/DESIGN.md` (주차별 계획 포함 — 작업 완료 시 체크 갱신).

## 문서 관리 원칙
- **작업을 마칠 때마다** 이 문서(구조·규칙 변경분)와 `docs/DESIGN.md`(체크리스트·시스템 설명)를 갱신할 것.
- 아래 "다음 후보" 섹션을 항상 최신으로 유지 — 끝낸 항목은 지우고, 새로 떠오른 아이디어를 우선순위와 함께 추가.

## 실행
- `npm install` — 최초 1회
- `npm run dev` — 개발 서버 (기본 포트 5175, `PORT` 환경변수로 변경 가능)
- `npm run typecheck` — 타입 검사
- `npm run build` — 프로덕션 빌드
- **자동화 스크립트** (`scripts/`, Playwright — 로컬은 설치된 Chrome 재사용): `npm run smoke`(부팅 스모크 — 배포 게이트와 동일), `npm run daily-ghost`(오늘의 던전 사서 실주행 → public/ghost/날짜.json, 멱등), `RUNS=n npm run balance-regression`(하드런 분포 vs 기준선), `npm run journal-data`(journal.html 데이터 블록·통계 타일 재생성). 워크플로: deploy-pages.yml(푸시마다 typecheck→build→**스모크**→배포), daily-ghost.yml(00:15 KST 크론 — 실주행 커밋+직접 배포), balance-night.yml(03:30 KST 크론 — 드리프트 시 이슈 자동 생성).
- **디버그 층 이동: Shift+D** (Esc 닫기) — 빠른 버튼(1·5·10·…·100층) + 직접 입력, 이동 시 체력 회복. DEV에서는 항상, **배포판에서는 `?debug` 쿼리 필요** (예: …/dungeon100/?debug — 일반 방문자 오작동 방지 게이트, 2026-07-13). `e.code` 기반이라 한/영 입력 상태 무관. 일반 `d`는 이동 키라 Shift 조합 사용, 이동 키는 Shift 조합 무시.
- **디버그 보물: Shift+P** (run phase에서만, 같은 `?debug` 게이트) — 보물방 클리어(전설)와 동일 보상: 랜덤 아이템 3개 + 완전 회복 + 황금 잔광·전설음. 흐름 끊김 없이 즉시 지급(quiz/memory 화면 안 거침), 누를 때마다 다른 아이템(grantRewards에 Math.random 시드 전달).
- **디버그 코인: Shift+M** (같은 `?debug` 게이트, phase 무관) — 코인 +100. 대장간(메타 강화) 테스트용, 코인은 localStorage라 유지됨.
- **자동 시연 (상시 노출, 약 2분)** — 타이틀 세로 메뉴 네 번째 "🎬 자동 시연 보기" 버튼 (2026-07-24 ⚡빠른 시작이 2번째로 들어오며 한 칸 밀림). **2026-07-19 `?demo` 게이트 해제** — **이 시연이 최신 콘텐츠를 계속 보여 주는 상시 쇼케이스** 역할(README도 이 버튼을 안내). 타이틀 기본 포커스는 "모험 시작"이라 Enter로 시연이 잘못 시작될 일은 없음. 투어: 마을 산책 → **촌장 대화(가짜 선택 개그 "다른 방법은 없나요?"→"없어.")** → 1층 전투 → 보물 3연속(16종·희귀도·시너지) → **낡은 제단(수락까지 시연 — 체력 바치고 보물 획득)** → 두 문 달리기 → 몬스터 아레나 → **몬스터 하우스(층=시드라 배치가 결정적 → `findFloor`로 있는 층을 찾아 점프)** → **찢어진 페이지(수락 = 2개 층 건너뛰기)** → 45·75층(테마·안개) → **56층 소녀 찻자리(공주 장면 — goTown(girlScript(false))로 직접 진입, girlMet 저장 안 함 = 관람자 세이브 오염 방지)** → 85층 폐허 마을 → 30층 보스 → 일일 던전 예고 → 끝 화면(다시 보기/직접 플레이). 시나리오 드라이버(`src/demo/useDemoDriver.ts`)가 합성 키보드 이벤트(useMoveInput/useSteer가 window 키 이벤트 기반, `code`만 채워 표준 메뉴의 `e.key` 핸들러와 충돌 없음)로 걷고, 장면 전환은 App이 넘겨준 조작 프리미티브(DemoActions — village/dungeon/jump/treasure/altar/secretDoor 등) 직접 호출 — DEV 훅 불사용이라 배포판에서도 동작. 오버레이는 1.5초 보여주고 자동으로 넘김(포털·마을 문·방 이벤트는 거절), 명시적으로 수락해야 하는 장면은 `clickChoice`/`clickPrimary`로 직접 클릭. 자막 `.demo-caption`, 종료 버튼, 끝 화면 `.demo-end`. **세이브 격리**(2026-07-19): 시연은 실제 게임을 그대로 플레이하므로 시작 시 `suspendPersistence(true)`(`src/lib/store.ts`)로 **localStorage 쓰기를 끄고**(읽기·React 상태는 정상 → 시연은 자연스럽게 진행), 나가는 모든 경로를 **리로드로 통일**해 원래 세이브·화면으로 복귀한다 — 종료 버튼·"직접 플레이"는 그냥 리로드, "다시 보기"는 `d100-demo-auto` 세션 키를 심고 리로드해 자동 재생(처음부터). 그래서 시연 중 얻은 코인·기억·최고 층·일일 기록이 세이브에 남지 않고, 시연 뒤 일반 플레이는 저장이 정상 복구된다(모듈 플래그가 리로드로 초기화). 음소거(`d100-muted`)는 사용자 설정이라 sound.ts가 직접 저장 — 격리 대상 아님. **주의: 헤드리스(숨김 탭)에서는 크롬 집중 스로틀링으로 드라이버의 setTimeout이 죽어 시연 전체 재생 검증이 불가** — 개별 장면은 MessageChannel 틱 + `__pump`로 검증할 것.

## 원칙
- UI 문구는 한국어. 백엔드 없음 — localStorage만.
- 외부 에셋 금지 — 절차 생성 지오메트리 + 캔버스/이모지 + Web Audio 합성으로 해결. 부득이 추가 시 출처·라이선스를 문서에 기록.
- 게임 로직은 렌더와 분리된 순수 함수 지향 (밸런스 자동 시뮬레이터가 로드맵에 있음 — 헤드리스로 돌릴 수 있어야 함).
- 커밋 기록이 곧 개발 과정의 증빙 — 작업 단위마다 의미 있는 한국어 커밋 메시지로 커밋하고, **작업이 끝나면 항상 바로 push** (main 푸시 = Pages 자동 배포, 2026-07-17 지시).
