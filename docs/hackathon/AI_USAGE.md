# 백층 던전 (Dungeon 100) — AI 활용 기술 문서

> NAN 2026 (NHN Game × AI Hackathon) 사전과제 제출물 4
> 제출자: 김학현 (개인 참여)

## 1. 사용한 AI 도구

| 도구 | 용도 |
|------|------|
| **Claude Code** (Anthropic) | 주력 개발 도구. Claude 모델 기반 CLI 코딩 에이전트로, 게임 기획 컨설팅·설계·코드 작성·스토리 집필·사운드 설계·자동 플레이 검증·배포·문서 작성까지 개발 전 과정에 사용 |

이미지·사운드 생성 AI는 사용하지 않았습니다 — 그래픽은 절차 생성 지오메트리, 사운드는 Web Audio 합성 코드이며, 이 코드들 역시 Claude Code가 작성했습니다.

**이 게임은 장르 선정부터 배포까지 전 과정을 AI와의 대화로 만든 게임입니다.** 코어 루프는 이틀 만에 완성했고, 이후 약 일주일에 걸쳐 스토리 완결·몬스터 아레나·시간의 흐름 연출을 더했습니다. 과정은 저장소 커밋 기록에 그대로 남아 있습니다.

## 2. AI 활용 구조 (개발 방법론)

### 2.1 게임 기획 자체를 AI와 — 장르 컨설팅
"AI로 잘 만들 수 있는 게임"을 AI에게 물어 장르를 정했습니다. Claude Code가 "복잡성이 에셋이 아니라 코드에 있는 게임(절차 생성·검증 가능한 규칙·텍스트 콘텐츠)"이라는 기준을 제시했고, 그 기준에 가장 맞는 **절차 생성 로그라이크**를 함께 선정했습니다.

### 2.2 CLAUDE.md — 지속 컨텍스트 문서
프로젝트 루트의 `CLAUDE.md`에 아키텍처·게임 규칙·세계관 정답지·검증 방법을 유지하여, AI가 매 세션 일관된 맥락에서 작업하도록 했습니다. 예: "던전=쓰이다 만 책, 촌장=작가, 100층 문=뒤표지"라는 세계관 정답지를 문서화해 두고, 모든 로어·대사가 여기로 수렴하도록 했습니다.

### 2.3 AI가 만든 게임을 AI가 직접 플레이하며 검증 ★
이 프로젝트의 가장 특징적인 AI 활용입니다. Claude Code가 **자기가 만든 게임을 스스로 플레이해서 검증할 수 있도록 검증 하네스를 직접 설계·구현**했습니다:

- `?rafshim` — 숨김 탭에서 멈추는 rAF·ResizeObserver를 대체하는 개발용 심
- `window.__pump(n)` — 브라우저 타이머 스로틀링과 무관하게 게임 프레임을 동기적으로 밀어주는 결정적 스텝 실행기
- `window.__d100`(던전) / `__d100run`(두 문) / `__d100arena`(몬스터 아레나) / `__d100town`(마을) / `__d100app`(층 점프) — 순간이동·상태 조회·보스 처치용 디버그 훅 (프로덕션 빌드에서 자동 제외)

이 하네스로 AI가 "상자 접촉 → 두 문 달리기 → 정답/오답 → 보상/빈손 → 회상 씬 → 5층 마을 문 방문 → 몬스터 티어 변화"까지 **전체 게임 플로우를 자동 플레이로 재현·확인**한 뒤에만 배포했습니다. 사람은 실기기에서 조작감·연출만 최종 검수했습니다.

### 2.4 자매작의 인게임 이식 (크로스오버)
같은 제작자의 자매작 [두 문 러너](https://github.com/Hakhyun-Kim/door-runner)(플레이: https://hakhyun-kim.github.io/door-runner/ — **역시 Claude Code로 개발**)의 핵심 메커닉(정답 문을 몸으로 통과)을 이 게임의 보물상자 미니게임으로 이식했습니다. AI가 기존 게임의 트랙 비율·조작·연출 감각을 참조해 새 코드베이스에 재구현했으며, **두 문 러너의 게임플레이는 이 백층 던전 안에 이미 통합되어 있습니다** (별도 실행 없이 보물상자에서 그대로 플레이). 이번 해커톤 제출작은 두 게임이 하나로 합쳐진 백층 던전입니다.

## 3. 주요 프롬프트 및 지시 사항

실제 개발 대화에서 사용한 지시들입니다 (원문 요약).

### 3.1 장르 선정
> "게임 스케일이 좀 작아 보이는데, AI로 잘 만들 수 있는 게임은 어떤 게 있을까?"

### 3.2 크로스오버 미니게임의 탄생
> "보물상자나 아이템 얻는 곳을 들어가면 두 문 러너를 하나 풀고 아이템을 얻게 하면 어떨까? 아이템을 얻으면 시각적으로 확실하게 변화를 주고, 타격했을 때도 화면에 확실하게 효과를 줘."

> "두문 러너의 전체를 가져왔으면 — 지금은 선택만 하는데, 독립적인 게임이 돌아서 성공해야만 아이템을 가져올 수 있게. 통일성을 위해서 아이템을 먹으려면 무조건 두 문 러너 게임을 통과해야 함."

> "미니게임을 좀 더 길게 해주고, 길수록 아이템이 좋아지면 좋겠어." → AI가 푸시-유어-럭(문마다 "받고 나가기 vs 더 달리기" 선택, 3연속 완주 = 전설 보물) 구조를 제안·구현

### 3.3 스토리·세계관
> "게임에 스토리가 있으면 좋겠어. 전통 RPG처럼 마을에서 시작 → 퀘스트 받고 → 던전 입장. 어제까지는 한국에서 2026년에 살던 대학생 → 던전 책을 보다가 → 자고 일어나니 책에서 보던 마을… 게임이니까 내가 선택해서 무언가를 하는 느낌 (사실 선택은 하나밖에 없지만)."

> "보물을 먹으면 → 회상 씬. 그때 먹은 라면이 맛있었어, 친구들하고 재미있었어, 부모님과 여행, 첫사랑 등. 층을 내려갈 때는 → 벽에 글귀. 여긴 어딜까? 꿈일까, 시간의 틈일까? 내려갈수록 진중하고 힌트가 될 만한 내용. 5개의 층마다 마을로 돌아가는 문이 생기고, 마을 사람들이 아이템이나 힌트를 줌."

기획 의도만 주면 AI가 세계관 정답지(던전=책)를 설계하고, 인트로 6장(인터랙티브 퀴즈 포함)·마을 대화·기억 12개·벽의 글귀 26개·엔딩 2종+에필로그를 그 정답지에 수렴하도록 집필했습니다.

### 3.4 사운드·난이도
> "사운드 추가. 게임의 꽃은 사운드. 사운드 직접 생성. 초등학교 버전이라 난이도 선택이 있으면 좋을 듯 — 초등학교 던전/어른 던전. 문제 수준을 맞춰줄 필요가 있음."

→ Web Audio 합성 효과음 20종 + 절차 생성 BGM 5트랙(파일 0개)과 던전 입구 난이도 분기(문제 생성 티어 분리)를 구현.

### 3.5 지시 사항의 공통 원칙
- 사람은 **기획 의도와 재미의 방향**을 말하고, 설계·구현·밸런스·검증·배포는 AI에 위임
- 버그·어색함은 현상 그대로 서술 — 원인 진단은 AI가 수행
- 반복될 규칙과 세계관은 CLAUDE.md에 승격하여 영구화

## 4. AI 산출물 검증 방법

| 대상 | 검증 |
|------|------|
| 게임 플로우 전체 | AI 자동 플레이 (검증 하네스로 상자·미니게임·마을·층 이동·난이도 분기 전 경로 재현) |
| 코드 | `tsc --noEmit` 타입 검사 + CI(빌드·타입 검사 통과 시에만 배포) |
| 문제 생성기 | 난이도별 출제 범위를 자동 플레이로 표본 확인 (초등 1층 "4−2" vs 어른 1층 "47−21") |
| 스토리·문구 | 사람이 톤·재미 검수 |
| 조작감·연출·사운드 | 사람이 실기기(모바일/PC)에서 최종 검수 |
