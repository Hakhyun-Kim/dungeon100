# 백층 던전 — 설계 문서

> NAN 2026 사전과제 **제출작(확정)**. 자매작 「두 문 러너」의 두 문 달리기를 보물상자 미니게임으로 통합.

## 컨셉

**"매판 새로 만들어지는 던전을 뚫고 100층까지."**
탑다운 3D 로그라이크 — 절차 생성 던전 + 자동 조준 전투 + 층마다 보상 드래프트로 빌드를 쌓는 런 기반 게임. 죽으면 처음부터, 하지만 매판 다른 맵과 다른 빌드.

### 핵심 재미 기둥
1. **탐험** — 층마다 다른 방 구조, 깊어질수록 넓고 위험해지는 던전
2. **빌드** — 3택 1 보상의 조합 (멀티샷×연사×사거리 같은 시너지)
3. **압박** — 회복이 귀하다. 체력 관리와 "더 내려갈까"의 긴장

## 스토리 (v0.6)

**"책 속으로 떨어진 대학생의 귀환 대작전."**
2026년 한국의 대학생이 『백층 던전의 비밀』을 읽다 잠들었는데, 눈을 떠 보니 책 속 마을. 돈도 힘도 없으니 할 수 있는 건 던전 탐험뿐 — 100층 끝의 '집으로 가는 문'을 향해 내려간다.

- 인트로 5장(건너뛰기 가능) → 마을에서 촌장과 대화 → 퀘스트 수락 → 던전 입장.
- **모든 진행이 선택**: 퀘스트 수락도, 동굴 입장도, 다음 층 포털도 플레이어가 고른다 (선택지가 사실상 하나뿐인 건 개그 — "혹시 다른 방법은?" → "없어.").
- 포털에서 "아직"을 고르면 그 층에 남고, 포털에 다시 닿으면 다시 물어본다.
- 게임오버 → 여관에서 부활 → 바로 재도전 or 마을(촌장 대화·대장간) 경유. **죽음도 세계관의 일부** — 죽을수록 "아프지 않다 → 페이지 넘어가는 소리 → 책이 나를 다시 쓴다"로 위화감이 커지는 로어가 표시되고, 코인·영구 강화는 유지된다 ("이야기도, 이어진다").
- **보물 = 되찾는 기억**: 보물을 얻을 때마다 2026년 한국의 기억(라면, MT, 첫사랑, 강아지…)이 하나씩 돌아온다 (12개, localStorage `d100-mem`에 진행 저장, 타이틀에 "되찾은 기억 N개" 표시). 첫사랑 기억에 『백층 던전의 비밀』 떡밥.
- **층 하강 = 벽의 글귀**: 층에 도착할 때마다 벽의 글귀(로어) 한 조각 — 얕은 층은 실없고(1층 주민의 몬스터 리뷰), 깊어질수록 진중한 힌트 (던전은 '쓰인' 것, 마을엔 아이가 없다, 100층 문 = 뒤표지, 작가가 마지막 장을 못 썼다…). 22층부터는 범용 글귀 순환.
- **세계관 정답지(스포일러)**: 던전 = 쓰이다 만 책. 마을 = 서문. 떨어진 자들 = 읽다 잠든 독자들. 촌장 = 마지막 장을 못 쓴 작가. 플레이어의 하강 = 마지막 장을 대신 쓰는 행위. 100층의 문 = 뒤표지(책을 덮는 곳) = 집.
- **5층마다 마을 문(책갈피)**: 던전 안에 마을로 가는 나무 문이 나타난다(왜인지는 아무도 모른다 — 촌장: "책갈피 같지 않니?"). 방문하면 층 유지 상태로 걸어다니는 마을 — 촌장이 층 티어별로 세계관을 한 걸음씩 고백(20층 = 작가 고백), 니나 수프(완전 회복), 무크 대장간. 문은 1회용, "나중에"를 고르면 남아 있음.
- **시간의 흐름**: 깊이 = 이야기의 진행. 마을은 20층 단위 5개 시절(고요한 밤→가을바람→습격의 흔적→방벽의 시간→폐허와 새벽)로 변하고 NPC 대사도 시절을 따라간다. 던전은 10층 단위 테마 10종(보랏빛 서장→…→금빛 마지막 장)에 깊이 비례로 안개가 짙어진다.
- **56층의 소녀 '여백' = 쓰다 만 공주**: 14·28·42·49층 흔적(복선) → 56층 만남. 42층 초대장을 봤으면 첫인사·답례 아이템 분기. 여백은 작가 노트 속 이름 빈칸의 「공주님」 — 함께 엔딩에서 촌장이 이름을 지어 주기로 한다.
- **엔딩과 에필로그**: 100층 황금 문에서 혼자/함께 선택 → 여백을 만났으면 추가 장면 → 전 엔딩 공통 10년 후 에필로그 (성장 회수 + 『2권에서 계속』 예고).

## 현재 구현 (v0.9 — 스토리 완결 + 시간의 흐름 + 걸어다니는 마을 + 코어 루프 + 두 문 달리기 + 몬스터 아레나 + 타격감)

- 시드 고정(mulberry32) 절차 생성: 방 5~9개 배치 + 폭 2 L자 복도로 연결 보장. 층 번호 = 시드.
- 이동: WASD/방향키 + 터치 드래그 가상 스틱, 그리드 벽 충돌.
- 전투: 사거리 내 최근접 적 자동 조준, 투사체 인스턴싱. 적은 어그로 범위 내 추격 + 접촉 피해.
- **보물상자 미니게임 (층당 1개) — 던전 입구에서 고른 모드에 따라 갈림**:
  - **수학 모드(🎒 초등 / 🧠 어른) = 두 문 달리기 (두 문 러너 인게임 재현, 최대 3연속 문)**: 상자에 닿으면 씬이 전환되어 **실제 두 문 러너**가 시작된다 — 자동 달리기 + 좌우 조작(화면 좌/우 꾹, ←/→)으로 정답 문을 몸으로 통과. 문 사이 벽은 막혀 있어 반드시 한쪽을 골라야 함. **푸시-유어-럭**: 문을 통과할 때마다 "보상 받고 나가기(아이템 = 통과한 문 수)" vs "더 달리기" 선택 — 3문 완주는 **전설의 보물**(아이템 3개 + 완전 회복), 도중에 오답 문에 부딪히면 💥 그동안의 보상까지 전부 빈손. 라운드가 깊어질수록 문제도 어려워짐(층 환산 +6/라운드). 미니게임 직후 0.9초 피격 자비.
  - **몬스터 모드(👹) = 몬스터 아레나 (GemArenaScene)**: 상자를 열면 수학 대신 오픈 아레나로 전환 — 우르르 몰려오는 무리(계속 재스폰)를 본체와 같은 자동 조준 전투로 뚫으며 바닥의 **보석 3개**를 몸으로 줍는다. 보석 하나 = 아이템 하나, 3개 완수 = 전설 보물(3개 + 완전 회복). **아레나 전용 체력**(100, 본체와 분리)이 0이 되면 쓰러지지만 본체는 무사 — **몇 번이고 재도전** 또는 그때까지 모은 보석만큼 받고 나가기. 무한 재도전이므로 처치는 코인/기록에 반영하지 않음(파밍 방지). 머리 안 쓰고 액션으로 보상을 얻는 갈래. **1층부터 4타입(추격/슈터/대셔/탱커) 전부 섞여** 나와 아래층 던전 전투를 미리 맛보게 하고 긴장감을 올린다(슈터 탄막 포함).
- **타격감**: 피격 적 흰색 번쩍+부풀기(인스턴스 색상), 넉백, 타격 스파크·처치 폭발 파티클(인스턴스 풀 160), 카메라 셰이크, 플레이어 피격 시 빨간 비네트.
- **획득의 시각화**: 멀티샷 → 궤도 구슬, 공격력 → 투사체 크기·색 3단계, 체력 → 몸집, 보물 → 황금 잔광. HUD에 현재 빌드 칩(아이콘×개수).
- 층 클리어(포털 접촉) → 보상 3택 1 (공격력/연사/이속/체력/사거리/멀티샷) → 다음 층.
- 난이도 곡선: 층수에 따라 방 수·적 수·적 체력·적 데미지·적 속도 상승. **5층 단위 몬스터 티어** — 모양(정육면체→옥타헤드론→콘→도데카→실린더→이코사)과 색(핑크→초록→파랑→보라→주황→서리)이 바뀐다 (30층 주기 순환). **스릴 램프(2026-07-18)**: "시작은 쉽게, 내려갈수록 가파르게" — 적 이동·탄속·연사·스폰 밀도가 전부 층 비례로 상승, 슈터는 8층+ 부채 2발·20층+ 3발, 위험 타입(슈터 2층+/대셔 4층+/탱커 6층+) 비중도 깊이 비례. 기준: 1층 8초 방치 HP −14(초보 안전) / 7층 방치 2.8초 사망 / 10층 보스방 방치 1.9초 사망 — 깊은 층은 계속 움직여야 산다.
- **보스 "페이지의 수호자"** (10층마다): 출구를 지키며 포털을 봉인. 추격 + 방사 탄막 10~14발(깊이별) + **저체력 광폭화**(발사 간격 단축) + 강한 접촉 피해. **hp 500+층×50** — 성장한 빌드도 탄막을 피하며 십수 초는 싸워야 하는 진짜 보스전. 처치 시 확정 보물 + 회복 + 봉인 해제. 100층 보스는 최종전 — 처치하면 황금 문(엔딩).
- **출구 수문장** (보스 없는 층, 4층부터): 포털 문 앞을 크고 진홍빛으로 빛나는 정예가 지킨다. 넓은 어그로·빠른 접근·매서운 돌진(넉백 면역)으로 **HP를 크게 깎아**(돌진 강타 `16+층×0.9`) 큰 데미지 숫자를 띄운다 — 캐릭터가 강해져도 문 앞의 긴장을 유지하는 장치. 봉인은 하지 않아(하드 게이트 X) 위험을 감수하고 지나갈지 처치할지는 선택.
- **엔딩 2종** (100층): 혼자 문을 열기(여운) / 촌장을 데리러 가기(진엔딩 — "그리고 두 사람은 함께 문을 나섰다"). 완주 통계 표시.
- 게임오버: 도달 층수 + 최고 기록(localStorage `d100-best`).
- **부활 체크포인트**: 5층마다 마을 문에 들르면 그 층이 부활 지점이 된다. 죽으면 1층이 아니라 **마지막으로 다녀온 마을에서 부활** — 주민들이 맞아 주고(장비·스탯 유지·완전 회복), 거기서 다시 던전으로 내려간다. 마을을 한 번도 안 들렀으면 기존처럼 1층부터. '주인공은 죽지 않고 다시 쓰일 뿐'이라는 죽음 로어와 직접 연결.
- **걸어다니는 3D 마을** (TownScene): 던전 입장 전·5층마다 방문·부활 — 모든 마을이 옛날 RPG처럼 직접 걸어다니는 광장이 된다. 우물·여관·대장간·촌장 집·나무. NPC(촌장/니나/무크)에 다가가면 대화, 던전 입구 아치로 걸어가면 내려간다(입장 시 난이도 선택, 방문·부활 시 이어서). 무크에게 말 걸면 대장간(영구 강화). 대사·회복·상점·세계관 회수를 걸어다니며 만난다. 56층 소녀는 여전히 특별 연출(별도 화면).
- 주인공 자동 조준은 일반 적뿐 아니라 **보스도 대상**으로 삼는다(예전엔 일반 적을 다 치우면 보스를 못 쐈음).

## 확장 방향

새 기능은 아래 3가지 기준을 모두 통과해야 채택한다:

1. **절차 생성 우선** — 외부 에셋 없이 코드(시드 난수·수식·캔버스·Web Audio)로 만들 수 있는가. 기존 시드 체계(층=시드)를 재사용하면 가산점.
2. **재미** — 코어 루프(탐험→빌드→압박)를 실제로 강화하는가. 장식이 아니라 의사결정이 늘어나는가.
3. **검증 가능** — 순수 로직이어서 밸런스 시뮬봇(simBot)으로 자동 측정할 수 있는가.

## AI 활용 스토리 (기술 문서용 메모)

- 절차 생성 + 시드 고정 → "같은 층 번호는 같은 맵" 재현성. 디버깅·밸런스에 유리.
- 밸런스 시뮬레이터: 게임 로직이 렌더와 분리된 순수 함수 지향이므로, AI가 작성한 자동 플레이 봇으로 수천 판을 돌려 난이도 곡선을 데이터로 튜닝 — "AI가 만들고 AI가 검증"하는 구조.
- 두문러너에서 확립한 패턴 재사용: 캔버스 텍스처 한글 렌더링, Web Audio 합성, rafshim 검증 심, localStorage 스토어.

## 기술 노트

- 지형·적·투사체·파티클 모두 InstancedMesh — 층 전체를 그려도 드로우콜 한 자릿수.
- `?rafshim`: 숨김 탭 검증용 심 — 타이머 구동 rAF(저부하) + ResizeObserver 폴링 폴리필(없으면 r3f가 캔버스 크기를 못 재서 WebGL 부팅 불가).
- DEV 전용 디버그 훅 — 브라우저 자동화로 AI가 전 플로우를 직접 플레이하며 검증 (밸런스 시뮬레이터의 기반): `__d100`(던전 teleport/state/hitBoss/killEnemies), `__d100run`(두 문), `__d100arena`(아레나), `__d100town`(마을), `__d100app`(층 점프). 디버그 단축키: Shift+D 층 이동 / Shift+P 전설 보물 / Shift+M 코인 +100 (DEV 항상, 배포판 `?debug`).
- 층 전환은 씬 컴포넌트 key 리마운트 (Canvas는 유지 → WebGL 컨텍스트 재생성 없음).
- 시뮬레이션은 useFrame 안에서 ref 기반 (React 상태는 HUD 이벤트만 — 피해/처치/층 이동).
- localStorage 키(전부 `d100-` 접두사): best(최고 층)·story(인트로)·mem(기억)·muted(음소거)·coins(코인)·meta(영구 강화)·deaths(사망 횟수)·traces(본 흔적 층)·girl(소녀 만남).
